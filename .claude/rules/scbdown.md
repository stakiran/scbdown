# scbdown specification
scbdown はマークアップ言語のフォーマットであり、一言で言えば Scrapbox と Markdown のハイブリットなフォーマットである。

この仕様書はフォーマットと文法を述べたものであり、要件を番号順に述べていく。

## 1: まずは Markdown を踏襲すよ

## 2: 段落内改行をサポートせよ
Markdown では段落内の改行をサポートしていなかった。

たとえば以下のmarkdownを考えよう:

```markdown
あああ
いいい

ううう
えええええ
おおお
```

レンダリングすると以下になる（段落内の改行がスペースになる）:

```
あああ いいい

ううう えええええ おおお
```

scbdown では段落内改行をサポートする。つまり、レンダリングすると以下になる:

```
あああ
いいい

ううう
えええええ
おおお
```

## 3: 複数行の空行をサポートせよ
Markdown では空行が 2 行以上あっても 1 行とみなされていた。たとえば以下の Markdown を考えよう:

```markdown
あああ



いいい
```

レンダリングすると以下になる（空行が3行あるはずなのに1行になる）

```
あああ

いいい
```

scbdown では空行一行一行を厳密に認識する。そのためレンダリングしても以下になる:

```
あああ



いいい
```

## 4: リストは 1-space indent とせよ
Markdown ではリストは `- ` または `* ` を用いていた。

scbdown では 1-space indent を用いる。

Markdown で書いたリスト1を示す:

```markdown
- リスト
- リスト
    - サブ
    - サブ
    - サブ
- リスト
    - サブ
        - サブのサブ
```

次にリスト1を scbdown で書いたリスト2を示す:

```scbdown
 リスト
 リスト
  サブ
  サブ
  サブ
 リスト
   サブ
    サブのサブ
```

## 5: 太字と斜体を廃止せよ
Markdown では `**太字**` と `*斜体*` をサポートしていたが、scbdown では廃止する。

## 6: 区切り線を新たにサポートせよ
Markdown で用いられる区切り線は廃止せよ。かわりに scbdown では行頭が `-` または `=` で始まる行を区切り行を用いる。

例:

```scbdown
# 見出し
段落1
段落1

段落2
段落2段落2段落2
ここも段落2

-

段落3。
区切りを入れてるよ。

---

段落4
 ここまで区切りを二つ入れたよ
  一つは段落1,2と段落3の間に
  もう一つは段落3と段落4の間に

# 複数行の区切り

ここからは複数行の区切りを見ていこう

=====
- 区切りの2文字目以降は何でもいいよ
- なのでこんな風に複数行にまたがった区切りもつくれるよ
====

上記は4行分の区切りを書いている。
```

なお区切りは、HTML でいう hr に相当するものである。区切り行の2文字目以降に書かれた文字はレンダリング時にはスルーされる。

## 7: リンク記法を新たにサポートせよ
Markdown ではリンク記法は `[text](uri)` であったが、これは廃止する。

scbdown では Scrapbox のリンク記法を用いる。

例:

```scbdown
内部リンク
 [あああ]
 これは`あああ.smd`へのリンクを示す

内部リンクとファイル名置換
 [#include "stdio.h"]
 これは`_include__stdio.h_.smd`へのリンクを示す
 ファイル名は Windows ファイルシステム上で有効でなければならず、無効な文字はすべて `_` に置換せよ
 また半角スペースも `_` に置換せよ

外部リンク
 [GitHub https://github.com/]
 [https://github.com/ GitHub]
 スペース区切りのテキストが複数与えられた場合、先頭または末尾に URL が含まれるはずである。
 この場合、URL をジャンプ先とし、それ以外の部分はすべて Text とみなす。
 ただし先頭または末尾に URL が見つからなかった場合は、内部リンクとして処理する

アイコン記法
 [xxx.icon]
 末尾が `.icon` で終わる場合はアイコン記法である。
 意味的には内部リンクと同等であるが、レンダリング時はページ xxx 内の最も上部にある画像をアイコンサイズで表示する。
 たとえば sta という人物ページをつくり、sta のアバターを貼っていた場合、`[sta.icon]` と書かれた部分をレンダリングすると sta のアバターがアイコンサイズで表示されるはずだ。

リンク記法のネスト
 リンク記法はネストできない。最初の `]` でリンクが閉じる。
 例: `[あああ[いいい]]` → リンクテキストは `あああ[いいい` となり、末尾の `]` はただの文字として扱う。
 エスケープもサポートしない。
```

## 8: 画像記法を新たにサポートせよ
Markdown の `![alt](url)` は廃止する。

scbdown ではリンク記法の `!` プレフィックス版を用いる。

例:

```scbdown
画像のみ
 ![https://example.com/img.png]

画像と代替テキスト（URL先頭）
 ![https://example.com/img.png 代替テキスト]

画像と代替テキスト（URL末尾）
 ![代替テキスト https://example.com/img.png]
```

リンク記法と同様に、スペース区切りで先頭または末尾に URL がある場合、URL を画像ソースとし、それ以外を代替テキストとする。

## 補足: Markdown から踏襲する要素
要件1「Markdown を踏襲する」に基づき、以下は明示的に廃止・上書きされていない限りサポートする:

- 見出し (`#`, `##`, `###` 等)
- インラインコード (`` ` ``)
- コードブロック (`` ``` ``)
- 引用 (`>`)

## 補足: コードブロック内の扱い
コードブロック (`` ``` ``) 内では scbdown 独自の文法(リンク記法、区切り線、1-space indent リスト等)はすべて無効化される。コードブロック内のテキストはそのまま表示する。
